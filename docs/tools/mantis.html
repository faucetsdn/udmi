Mantis is an automated diagnostic utility designed to consolidate run support bundles, evaluate test stability trends, and perform differential failure log analysis across the UDMI software stack.

Simplifying Log Triage

Testing UDMI sequences produces complex outputs distributed across several layers: the Sequencer validator, Pubber client simulator, MQTT broker, and cloud-side UDMIS infrastructure.

Analyzing a test failure or flaky behavioral variance manually is time-consuming:

  • Varying Time Triggers: Simple diffs are not effective because timestamps, sequence ID hashes, and runtime thread names vary on every execution.
  • High Concurrency Noise: Sifting through hundreds of lines of logs is necessary to locate a single timing offset or configuration mismatch.
  • Flakiness Isolation: It can be challenging to distinguish random execution flakes from consistent code regressions.

Mantis addresses this by automating the collection of run statistics, identifying flakiness distribution, and using differential matching algorithms to isolate failures.

Analysis Approach

Mantis handles test diagnosis by focusing strictly on high-signal behavioral diffs:

📋 Systematic Diagnostics

"Rather than checking static outputs, I map exact test execution time boundaries, locate successful sibling reference runs, filter out transaction noise, and link differences directly back to the source code files to show where execution diverged."

The Mantis Workflow

Support Bundle Aggregation Phase One

Mantis begins by aggregating run information. It pulls complete support bundles from local environments or CI test systems.

These bundles consolidate sequence execution stats, console logs, and configuration files, ensuring all diagnostic context is preserved.

CLI Command
bin/collect_stats 3 mantis
Output Directory
tools/mantis/out/test_bundles/
Suite Stability Evaluation Phase Two

The stability evaluator calculates run metrics across all collected bundles to identify failure patterns.

By evaluating failure rates and comparing logs against reference runs, it flags which tests are consistently failing versus which are flaky.

CLI Command
bin/evaluate_stability
Output Summary
tools/mantis/out/stability_report.md
Differential Triage Phase Three

The core triage component performs targeted analysis on a failing test.

It automatically discovers successful sibling runs of the same test, extracts strict time boundaries, strips variable execution hashes, and pinpoints the exact step where the failure occurred.

CLI Command
bin/diagnose system_mode_restart
Output Report
triage_analysis.md (inside test folder)

CLI Reference

Mantis processes are executed locally via shell scripts. Use these core commands in your workflow:

1. stats collector

Collects test run bundles. Specify the run count and the branch.

bin/collect_stats 3 mantis

2. stability evaluator

Analyzes all collected bundles to build stability models and flag flaky tests.

bin/evaluate_stability

3. differential diagnose

Runs differential analysis on a targeted test to produce a localized triage report.

bin/diagnose system_mode_restart
